## 1. 페이지 레이아웃 원칙

### 1.1 핵심 원칙

- **한 페이지, 한 목적**: 각 페이지는 하나의 주요 작업에 집중
- **Mobile First**: 작은 화면부터 설계, 점진적 확장
- **콘텐츠 우선**: 불필요한 UI 요소 제거
- **일관된 네비게이션**: 모든 페이지에서 동일한 네비게이션 구조

### 1.2 레이아웃 계층

```
Page Layout:
├── Navigation (Sidebar / Bottom Nav)
├── Header (페이지 타이틀, 액션)
├── Main Content
│   ├── Filters (선택)
│   ├── Content Area
│   └── Secondary Panel (선택)
└── Footer (선택)
```

## 2. 공통 레이아웃 패턴

### 2.1 리스트 페이지

**구조:**
- 상단: 필터/검색
- 중앙: 카드 리스트 (세로 스크롤)
- 하단: 네비게이션

**예시:** 홈 (문제 목록), 통계

### 2.2 상세 페이지

**구조:**
- 상단: 뒤로 가기 + 타이틀
- 중앙: 콘텐츠
- 우측 패널: 관련 정보 (Desktop)

**예시:** 문제 상세

### 2.3 폼 페이지

**구조:**
- 중앙 정렬
- 최대 너비 제한 (400-600px)
- 단순한 레이아웃

**예시:** 로그인, 설정

### 2.4 대시보드 페이지

**구조:**
- 그리드 레이아웃 (2컬럼 이상)
- 카드 기반 섹션
- 반응형 (Mobile: 1컬럼, Tablet+: 2컬럼)

**예시:** 통계

## 3. 플랫폼별 차이

### 3.1 Navigation

**Mobile:**
- 하단 네비게이션 (Bottom Nav)
- 4-5개 주요 메뉴

**Desktop:**
- 좌측 사이드바 (Sidebar)
- 확장 가능 (아이콘 ↔ 텍스트)

### 3.2 Modal / Sheet

**Mobile:**
- Bottom Sheet (하단에서 슬라이드)
- 전체 화면 모달 (중요한 액션)

**Desktop:**
- 중앙 모달 (최대 너비 제한)
- 오버레이 배경

### 3.3 Detail View

**Mobile:**
- 세로 스크롤 (단일 컬럼)
- 탭으로 섹션 전환

**Desktop:**
- 2컬럼 레이아웃 (메인 + 사이드)
- 고정 사이드 패널

## 4. 실제 구현 참조

### 4.1 페이지 코드

**실제 페이지 레이아웃은 다음 폴더를 참조하세요:**

```
app/(routes)/              # Next.js 페이지 (웹)
app/(tabs)/                # Expo 탭 페이지 (모바일)
components/layouts/        # 레이아웃 컴포넌트
```

### 4.2 CODEMAPS 참조

**프로젝트의 실제 페이지 목록과 구조:**

```
codemaps/frontend.md   # 자동 생성된 페이지 구조
```

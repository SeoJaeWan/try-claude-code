Profile Guide

공통규칙

  • 폴더 경로는 각 profile의 path 규칙을 따르며, 공통 네이밍은 recipe에서 정규화한다.

  • 함수 기본 스타일은 화살표 함수다.

  • 배열 이름은 복수형을 사용하고 List, Array suffix는 피한다.

  • 한 곳에서만 쓰는 타입은 로컬에 두고, 여러 곳에서 쓰면 types/common 또는 types/{domain}으로 올린다.

  • 퍼블리셔는 비즈니스 로직을 구현하지 않는다.

  • toggle, sidebar, modal 같은 UI 상호작용 state는 허용한다.

  • React TSX 파일은 전부 컴포넌트로 본다. page.tsx는 page.tsx를 유지하고, 그 외 컴포넌트 엔트리는 index.tsx를 사용한다.

  • 컴포넌트 경로는 마지막 components 세그먼트 아래에서 동일 규칙을 따른다. 예: components/common/{component}, src/components/common/{component}, features/{feature}/components/{domain}/{component}.

  • domain은 app/{domain}/page.tsx의 root page segment를 사용한다. 여러 page domain에서 공통이면 common을 사용한다.

  • 레거시 컴포넌트 경로가 현재 컨벤션과 다르면 기존 위치를 유지하지 말고 먼저 현재 컨벤션 경로로 마이그레이션한다.

  • 레거시 경로가 이미 현재 컨벤션과 같으면 경로는 유지하고 내부 구현만 수정한다.

Guide

목적

기존 퍼블리셔 UI 파일이 배치 규칙과 AST 규칙을 지키는지 검증한다.

입력규칙

  • 파일 경로 1개 이상을 positional 또는 files 배열 JSON으로 전달하거나, root 경로를 넘겨 해당 디렉터리 아래의 검증 대상 파일을 모두 탐색한다.

출력규칙

  • results 배열로 파일별 위반 목록을 반환한다.

주의사항

  • page.tsx는 프레임워크 엔트리 파일명이라 그대로 유지한다.

  • page.tsx가 아닌 React TSX 파일은 경로와 상관없이 index.tsx 엔트리를 사용한다.

  • 파일/폴더 세그먼트는 camelCase를 사용한다. 단, page.tsx와 index.tsx 같은 예약 엔트리 파일명과 Next 특수 세그먼트((group), [id], @slot)는 예외다.

  • components 세그먼트 아래 파일은 prefix와 무관하게 index.tsx와 배치 규칙을 함께 검증한다.

  • components 세그먼트 밖의 React TSX 파일도 컴포넌트 AST 규칙은 동일하게 검증한다.

  • 같은 파일 안의 JSX helper와 subcomponent는 금지한다.

Command

component

Generate a publisher UI component file
input: jsonmode: filerequired: name, pathoutput: {path}/index.tsx

Guide

목적

UI 컴포넌트 파일 껍질을 생성한다.

입력규칙

  • name과 path는 필수다.

  • path는 마지막 components 세그먼트 아래에서 components/common/{component} 또는 components/{domain}/{component} 패턴을 사용한다. prefix는 허용된다.

  • props는 value 또는 callback 멤버 배열로 전달한다.

이름규칙

  • 컴포넌트 이름은 PascalCase를 사용한다.

  • props callback은 on* 규칙을 따른다.

출력규칙

  • 출력 파일은 {path}/index.tsx 이다.

  • Props 인터페이스와 export default를 포함한다.

  • 컴포넌트 본문은 화살표 함수 형태를 유지한다.

배치규칙

  • 공통 컴포넌트는 */components/common/{component}를 사용한다.

  • 도메인 전용 컴포넌트는 */components/{domain}/{component}를 사용한다.

  • prefix가 있어도 마지막 components 세그먼트를 기준으로 같은 규칙을 적용한다. 예: src/components/common/button, features/cart/components/cart/cartSummary.

  • domain은 root page segment를 사용한다. 예: app/login -> login.

  • common은 2개 이상 root page domain에서 재사용될 때만 사용한다.

  • 레거시 경로가 현재 컨벤션과 다르면 기존 PascalCase 폴더나 top-level components/{Component} 경로를 유지하지 말고 현재 경로 규칙으로 이동한다.

  • 레거시 경로가 이미 현재 컨벤션과 같으면 경로는 유지하고 index.tsx 내부 구현만 수정한다.

안티패턴

  • dashboard 전용인데 components/common/dashboardLayout에 두는 것

  • 범용 카드인데 components/ProductCard 같은 PascalCase top-level 폴더를 유지하는 것

  • pattern으로 toggle 또는 기본 패턴을 선택할 수 있다.

이름규칙

  • state 이름은 is*Open 규칙을 기본으로 사용한다.

  • setter는 setIs*Open 규칙을 기본으로 사용한다.

  • handler는 handleToggle*, handleOpen*, handleClose* 규칙을 사용한다.

출력규칙

  • state 선언과 handler 껍질을 함께 반환한다.

주의사항

  • 비즈니스 상태가 아니라 UI 상호작용 상태에만 사용한다.

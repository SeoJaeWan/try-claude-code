# 관리자 페이지 인증 테스트 플랜

## 개요

Next.js 16 기반 웹 애플리케이션의 관리자 페이지 접근 및 인증 흐름에 대한 E2E 테스트 플랜이다. 현재 앱은 기본 Next.js 스타터 상태(`app/page.tsx`)이며, 관리자 인증 기능이 구현될 것을 전제로 테스트 시나리오를 설계한다.

**기술 스택:** Next.js 16.1.6, React 19.2.3, TailwindCSS 4, TypeScript 5

---

### 1. 관리자 로그인

#### 1.1 유효한 자격 증명으로 로그인

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 로그인 폼이 렌더링되었는지 확인한다 (이메일/아이디 입력란, 비밀번호 입력란, 로그인 버튼).
3. 유효한 관리자 이메일을 이메일 입력란에 입력한다.
4. 유효한 비밀번호를 비밀번호 입력란에 입력한다.
5. "로그인" 버튼을 클릭한다.

**기대 결과:**
- 관리자 대시보드(`/admin/dashboard` 또는 `/admin`)로 리다이렉트된다.
- 대시보드 페이지에 관리자 환영 메시지 또는 네비게이션이 표시된다.
- 세션/쿠키가 올바르게 설정된다.

#### 1.2 잘못된 비밀번호로 로그인 실패

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 유효한 관리자 이메일을 입력한다.
3. 잘못된 비밀번호를 입력한다.
4. "로그인" 버튼을 클릭한다.

**기대 결과:**
- 로그인 페이지에 머무른다.
- "이메일 또는 비밀번호가 올바르지 않습니다" 등의 에러 메시지가 표시된다.
- 비밀번호 입력란이 초기화된다.

#### 1.3 존재하지 않는 계정으로 로그인 시도

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 등록되지 않은 이메일 주소를 입력한다.
3. 임의의 비밀번호를 입력한다.
4. "로그인" 버튼을 클릭한다.

**기대 결과:**
- 로그인 페이지에 머무른다.
- 적절한 에러 메시지가 표시된다.
- 계정 존재 여부를 노출하지 않는 일반적인 에러 메시지를 사용한다 (보안).

#### 1.4 빈 필드로 로그인 시도

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 아무 입력 없이 "로그인" 버튼을 클릭한다.

**기대 결과:**
- 폼 유효성 검증 에러가 표시된다.
- 이메일과 비밀번호 필드에 "필수 입력 항목입니다" 등의 메시지가 나타난다.
- API 요청이 발생하지 않는다.

#### 1.5 잘못된 이메일 형식으로 로그인 시도

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 이메일 입력란에 `invalid-email` 같은 잘못된 형식을 입력한다.
3. 비밀번호를 입력한다.
4. "로그인" 버튼을 클릭한다.

**기대 결과:**
- "올바른 이메일 형식을 입력해주세요" 등의 유효성 검증 에러가 표시된다.
- 로그인 요청이 서버로 전송되지 않는다.

### 2. 인증 보호 (접근 제어)

#### 2.1 비인증 상태에서 관리자 페이지 접근

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 로그인하지 않은 상태에서 `/admin/dashboard` 로 직접 이동한다.

**기대 결과:**
- `/admin/login` 페이지로 리다이렉트된다.
- 관리자 대시보드 컨텐츠가 노출되지 않는다.

#### 2.2 비인증 상태에서 관리자 하위 페이지 접근

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 로그인하지 않은 상태에서 `/admin/users`, `/admin/settings` 등 관리자 하위 경로로 직접 이동한다.

**기대 결과:**
- 모든 관리자 하위 페이지에서 `/admin/login` 으로 리다이렉트된다.
- 보호된 데이터가 순간적으로라도 화면에 표시되지 않는다 (flash 없음).

#### 2.3 만료된 세션으로 관리자 페이지 접근

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인한다.
2. 관리자 대시보드가 정상 표시되는지 확인한다.
3. 브라우저 쿠키/세션 토큰을 삭제하거나 만료시킨다.
4. 페이지를 새로고침한다.

**기대 결과:**
- `/admin/login` 페이지로 리다이렉트된다.
- "세션이 만료되었습니다. 다시 로그인해주세요" 등의 메시지가 표시될 수 있다.

### 3. 로그아웃

#### 3.1 정상 로그아웃

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인하여 대시보드에 접근한다.
2. 로그아웃 버튼 또는 메뉴를 찾아 클릭한다.

**기대 결과:**
- `/admin/login` 페이지 또는 메인 페이지(`/`)로 리다이렉트된다.
- 세션/쿠키가 삭제된다.

#### 3.2 로그아웃 후 뒤로가기 버튼

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인하여 대시보드에 접근한다.
2. 로그아웃한다.
3. 브라우저의 뒤로가기 버튼을 클릭한다.

**기대 결과:**
- 관리자 대시보드가 다시 표시되지 않는다.
- `/admin/login` 페이지로 리다이렉트되거나 캐시된 보호 페이지가 노출되지 않는다.

#### 3.3 로그아웃 후 관리자 URL 직접 접근

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인하여 대시보드에 접근한다.
2. 로그아웃한다.
3. 주소창에 `/admin/dashboard` 를 직접 입력하여 이동한다.

**기대 결과:**
- `/admin/login` 페이지로 리다이렉트된다.
- 관리자 컨텐츠가 노출되지 않는다.

### 4. 관리자 대시보드 기본 기능

#### 4.1 대시보드 페이지 렌더링

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인한다.
2. 관리자 대시보드 페이지가 로드되는지 확인한다.

**기대 결과:**
- 대시보드 레이아웃이 올바르게 렌더링된다.
- 네비게이션 메뉴 (사용자 관리, 설정 등)가 표시된다.
- 관리자 이름 또는 역할 정보가 표시된다.

#### 4.2 관리자 네비게이션

**Seed:** `tests/seed.spec.ts`

**단계:**
1. 관리자로 로그인하여 대시보드에 접근한다.
2. 네비게이션 메뉴의 각 항목을 클릭하여 이동한다.

**기대 결과:**
- 각 관리자 하위 페이지로 정상 이동된다.
- 현재 활성 메뉴 항목이 시각적으로 구분된다.
- 페이지 전환 시 레이아웃이 유지된다.

### 5. 로그인 페이지 UI/UX

#### 5.1 로그인 페이지 초기 렌더링

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.

**기대 결과:**
- 로그인 폼이 올바르게 표시된다.
- 이메일 입력란에 포커스가 자동 설정된다.
- 비밀번호 입력란은 마스킹(`type="password"`) 처리되어 있다.
- 로그인 버튼이 표시된다.

#### 5.2 비밀번호 표시/숨기기 토글

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 비밀번호 입력란에 비밀번호를 입력한다.
3. 비밀번호 표시 토글 버튼(있는 경우)을 클릭한다.

**기대 결과:**
- 비밀번호가 평문으로 표시된다.
- 다시 토글하면 마스킹된다.

#### 5.3 Enter 키로 로그인 제출

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 유효한 자격 증명을 입력한다.
3. 비밀번호 입력란에서 Enter 키를 누른다.

**기대 결과:**
- 로그인 버튼을 클릭한 것과 동일하게 로그인 요청이 제출된다.
- 관리자 대시보드로 리다이렉트된다.

#### 5.4 로그인 중 로딩 상태

**Seed:** `tests/seed.spec.ts`

**단계:**
1. `/admin/login` 페이지로 이동한다.
2. 유효한 자격 증명을 입력한다.
3. "로그인" 버튼을 클릭한다.

**기대 결과:**
- 로그인 처리 중 로딩 인디케이터가 표시된다.
- 로그인 버튼이 비활성화되어 중복 제출이 방지된다.
